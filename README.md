# 🛡️ DocGuard (文档守卫)

> 让法律文书脱敏像魔法一样简单，且排版一丝不苟。

**DocGuard** 是一款专为法律、财务及办公场景打造的**原生文档脱敏工具**。它不仅能通过大语言模型 (LLM) 智能找出隐私实体，还能在**完全保留原文件排版、字体、样式和印章**的前提下，对原生 Word、PDF、Excel、PPT 和 Markdown 文件进行精准的脱敏与打码！

---

## ✨ 核心特性

- 🎯 **原生样式零破坏 (NER + Redaction + OCR)**：
  摒弃了传统的"让大模型重写整段话"导致的排版灾难。DocGuard 仅让大模型扮演"找茬员"（实体识别 NER），随后使用 Python 库在底层对文件进行精确的"坐标遮挡 (PDF)"或"片段替换 (Word/PPT)"。加粗、字号、表格、甚至**红头印章**均完美保留！
  *(新功能：对于纯图片扫描件，自动启用轻量级急速 OCR 进行视觉识别，并计算原图坐标进行特种狙击式打码！)*

- 🔄 **全新！可逆脱敏模式（Reversible Anonymization）**：
  专为需要"脱敏→交给在线 AI 处理→还原"工作流设计。脱敏时用结构化占位符（`[[P001]]`、`[[ORG001]]` 等）替换敏感实体，同时生成 `mapping.json` 记录映射关系；拿回 AI 处理后的文档，一键还原为包含真实姓名的完整文件。全程隐私零泄漏。

- 🧠 **NER 引擎增强**：
  LLM 实体识别 + 正则兜底（手机号、身份证号自动补采）+ 噪声过滤（剔除个位数假金额）+ 空白归一化（解决跨行人名被拆分导致漏提取的问题）。

- 🔓 **灵活兼容任何大模型**：
  基于标准的 OpenAI API 接口协议通讯。既支持完全离线的本地部署（Ollama、LM Studio），实现敏感案卷**零泄漏、不触网**；也支持直接接入云端大模型（DeepSeek、通义千问、OpenAI）。

- 📦 **全能格式支持 (含纯图片)**：
  - 📄 **Word (`.docx`)**：精准替换 Run 级别文本，复杂版式丝毫不乱。
  - 📕 **PDF (`.pdf`)**：纯文本 PDF 与**纯图片扫描件 PDF** 均完美支持！基于几何坐标的纯黑遮挡 (Redaction)，印章完美存活。
  - 🖼️ **图片 (`.jpg` / `.png`)**：自动将其转为同等画质的脱敏 PDF，原样打码输出。
  - 📊 **Excel (`.xlsx`)**：工作表与图表无损保留，精准替换单元格。
  - 📽️ **PPT (`.pptx`)**：支持幻灯片内的文本框、形状智能替换。
  - 📝 **纯文本 (`.txt` / `.md`)**：轻量级字符替换。

- 📝 **独家双轨输出机制**：
  文件处理完成后，除了生成一份"完美保留原排版"的格式文件（用于人类阅读与交付）外，还会**自动附赠一份同名的纯文本 `.md` 文件**。不仅能剥离冗余排版，还能极大地节省 Token，专供下游大模型（如 ChatGPT/DeepSeek）做进一步的案情总结与财报分析。

- ⚡ **秒级增量缓存与拦截网**：
  - **指纹缓存**：自动记录已脱敏的文件指纹，重复文件秒级跳过，绝不白白浪费您的 API Token 和时间。
  - **质量兜底校验**：内置防御性正则检测引擎，自动扫描最终文本。若发现疑似漏提的百家姓或法律称谓，会在终端发出高亮警告，辅助人工安全复核。

- 🚀 **一键开箱即用**：
  专为非技术人员设计的"沙盒机制"。Mac 用户双击 `.command` 脚本，Windows 用户双击 `.bat` 脚本即可自动安装隔离的虚拟环境，**绝不污染你的系统全局配置**。

## 🔄 可逆脱敏工作流

这是 DocGuard 的核心功能，专门解决"需要把文件交给在线 AI（如 ChatGPT、Claude）处理，但又不想泄漏当事人隐私"的场景。

### 使用场景举例

> 你有一份张三起诉李四的起诉状，想让在线 AI 帮你写代理词，但不想上传真实当事人信息。

**第一步：可逆脱敏**（菜单选 2）

把原始文件放进 `input/`，DocGuard 把敏感信息替换为占位符，同时生成映射表：

```
张三  →  [[P001]]
李四  →  [[P002]]
北京某某律师事务所  →  [[ORG001]]
```

输出到带时间戳的 session 文件夹：
```
output/
  session_20260625_143022/
    脱敏_起诉状.docx        ← 原格式脱敏文件
    脱敏_起诉状.md          ← 纯文本版（喂给在线 AI 用，省 Token）
    mapping.json            ← 映射表，妥善保管
```

**第二步：交给在线 AI 处理**（DocGuard 不参与）

把脱敏后的 `.md` 文件交给任意在线大模型，让它写答辩状、分析案情、生成审查提纲等。AI 生成的文件里会带着占位符（可能是 `[[P001]]` 或 `P001`，DocGuard 都能识别）。

**第三步：还原**（菜单选 3）

把 AI 生成的文件放进 `restore/` 文件夹，选择对应的 session，DocGuard 自动把占位符还原回真实信息：

```
restore/                     ← 把 AI 生成的文件放这里
  答辩状.md

output/
  session_20260625_143022/
    还原_答辩状.md           ← 还原后的完整文件
```

支持还原的格式：`.md` / `.txt` / `.docx` / `.pdf` / `.xlsx`。

## 🛠️ 快速开始

### 1. 前置准备
1. 确保已安装 Python 环境。
2. 准备一个 **大语言模型 (LLM) API 服务**：
   - **本地离线方案 (推荐，保护隐私)**：使用 [LM Studio](https://lmstudio.ai/)、[Ollama](https://ollama.com/) 或 vLLM 在本地运行模型。
   - **云端在线方案**：如果文件不涉密，也可以直接使用 DeepSeek、通义千问、OpenAI 等云端 API。
3. **关于大模型的选择**：
   - **最低要求**：模型需要具备良好的"指令遵循 (Instruction Following)"能力，能够稳定输出 JSON 格式数据，并且中文命名实体识别 (NER) 能力达标。一般建议选择 **7B 规模及以上** 的参数量。
   - **推荐模型**：强烈推荐使用 **Qwen 2.5 (通义千问) 7B 或 14B** 系列模型，它在零样本实体抽取上的表现极其稳定且精准。

### 2. 运行工具
1. 克隆或下载本仓库代码。
2. 进入 `docguard` 文件夹。
3. **Mac 用户**：直接双击运行 `一键DocGuard脱敏.command`。
4. **Windows 用户**：直接双击运行 `一键DocGuard脱敏.bat`。

> 💡 **提示**：无论您使用的是 Mac 还是 Windows，**首次运行时，脚本都会自动在后台创建 `.venv` 虚拟环境并下载所需依赖**，这可能需要几分钟的时间，请耐心等待终端的进度条跑完。

### 3. 选择模式

程序启动后会显示交互式菜单：

```
==================================================
  DocGuard 文档守卫
==================================================
  1. 普通打码（不可逆）
  2. 可逆脱敏（生成 mapping）
  3. 还原文档（从 mapping 恢复）
==================================================
请输入选项编号（1/2/3）：
```

启动时程序会自动创建三个工作文件夹：
- `input/` — 放待脱敏的原始文件
- `output/` — 存放脱敏结果（按 session 分文件夹）
- `restore/` — 放待还原的文件（大模型生成的答辩状、审查提纲等）

- **选项 1（普通打码）**：将敏感信息替换为 `[脱敏]`，不可逆，输出至 `output/`。
- **选项 2（可逆脱敏）**：用占位符替换敏感信息，生成 `mapping.json`，输出到 `output/session_时间戳/`，同时生成 `.md` 纯文本版供下游 AI 使用。
- **选项 3（还原）**：把 `restore/` 中的文件（含占位符）还原为真实信息，输出到对应 session 文件夹。

### 4. 普通打码流程
1. 将需要脱敏的文件拖入 `input/` 文件夹。
2. 运行程序，选择 **1**。
3. 处理完毕后，脱敏好的文件将安静地躺在 `output/` 文件夹中。

### 5. 可逆脱敏 + 还原流程
1. 将需要脱敏的文件拖入 `input/` 文件夹。
2. 运行程序，选择 **2**，脱敏文件 + `mapping.json` + `.md` 纯文本版输出到 `output/session_时间戳/`。
3. 把脱敏后的 `.md` 文件交给在线 AI 处理，拿回生成的文件（答辩状、审查提纲等）。
4. 把 AI 生成的文件放入 `restore/` 文件夹。
5. 再次运行程序，选择 **3**，选择对应 session，程序自动还原占位符，输出到 session 文件夹。

## ⚙️ 进阶配置

程序通过读取环境变量或直接修改代码来配置你的大模型后端。你可以用任意文本编辑器打开 **`ner_engine.py`** 顶部，修改以下默认值：

- `DOCGUARD_API_BASE` (默认: `http://localhost:1234/v1`，这是 LM Studio 的默认地址。若用 Ollama 通常是 `http://localhost:11434/v1`，或填入云端 API 地址)
- `DOCGUARD_API_KEY` (默认: `lm-studio`。若使用真实的云端 API 请填入真实 Key)
- `DOCGUARD_MODEL` (默认: `qwen2.5-14b-instruct-1m`。请填入你实际运行的模型名称)

## ⚠️ 局限性与注意事项

- 脱敏是一项严谨的工作，大模型难免会有漏网之鱼，**交付给客户前请务必进行人工复核检查！**
- 可逆脱敏模式中，请妥善保管 `mapping.json`，丢失后无法还原。
- 可逆脱敏支持 `.docx` / `.pdf` / `.xlsx` / `.txt` / `.md` 全格式，还原同理。
- 还原时占位符兼容两种格式：`[[P001]]`（带括号）和 `P001`（不带括号），应对大模型可能清理掉 markdown 标记的情况。
- 交给在线 AI 处理时，建议在提示词中注明"保持所有 `[[...]]` 格式的标记原样输出"，以确保占位符完整保留（即使 AI 去掉了括号，DocGuard 也能兜底还原）。
- PDF 可逆脱敏使用 redaction 真正删除底层文字后叠加占位符，还原时为视觉覆盖，字体可能与原文略有差异。
